1.数组 vs 链表的区别
存储方式不同：数组--连续内存空间 链表--非连续内存，通过指针连接节点
大小灵活性：数组--大小固定（静态数组），或需整体复制扩容（动态数组）。链表--动态			增删节点，无需预先确定大小。
访问效率：数组--支持随机访问 O(1) 链表--只能顺序访问 O(n)

2.栈和队列的应用场景Application Scenarios for Stacks and Queues
栈：特点是后进先出，典型应用场景有：函数调用与递归（系统用“调用栈”保存函数的返回地址、参数和局部变量，每调用一个函数就入栈，返回时就出栈。）/ 编译器中括号匹配检查（遇到左括号入栈，遇到右括号出栈匹配）/ 深度优先搜索(DFS) -- 用栈保存待访问的节点
Stack: Characterized by Last-In-First-Out (LIFO) behavior.
Typical applications include:
Function calls and recursion (systems use a “call stack” to store function return addresses, parameters, and local variables; each function call pushes onto the stack, and returns pop off the stack). / Bracket matching in compilers (push left brackets onto the stack, pop right brackets to match)/ Depth-first search (DFS) — using a stack to store nodes to be visited
队列：特点是先进先出，典型应用场景有：广度优先搜索（BFS）-- 用队列保存待扩展的节点。/ 操作系统中的进程调度（如就绪队列）
Queues: Characterized by First-In-First-Out (FIFO) behavior. Typical applications include:
/ Breadth-first search (BFS) — using a queue to store nodes to be expanded. / Process scheduling in operating systems (e.g., ready queue)

3.常见的排序算法/时间复杂度 O(n), O(log n) 举例Sortieralgorithmen/Zeitkomplexität
排序算法	平均时间复杂度Average time complexity	空间复杂度	应用场景
冒泡排序Bubble Sort	O(n2)n squared	O(1)

冒泡排序Bubble Sort：Starting from the leftmost position, continuously compare adjacent pairs of numbers. If the left number is greater than the right one, swap them.
选择排序Selection Sort：Each time, select the smallest number from the unsorted digits and place it in its correct position at the front.
插入排序Insertion Sort：At the beginning, there is only one number. New numbers are continuously inserted into the correct position at the front of the already sorted sequence.
快速排序Quick Sort：Choose a number as the reference point. Place smaller numbers to the left and larger numbers to the right. Then repeat the process on both sides.
时间复杂度O(n2)>O(n log n)>O(n+k)

The significance of the Turing machine lies in its definition of “computability.”

10.死锁是什么？如何避免？What is deadlock? How can it be avoided?
定义：一个进程集合是死锁的，如果集合中的每个进程都在等待某一事件的发生，而只有集合中的另一个进程才能导致该事件的发生。
如何避免：银行家算法
Definition: A set of processes is deadlocked if each process in the set is waiting for an event to occur, and only another process in the set can trigger that event.
How to avoid: Banker's Algorithm

Q4: 为什么RAG可以预防虚假信息？Warum kann RAG Falschinformationen vorbeugen?
传统模型的任务是根据输入，预测下一个最有可能出现的词。RAG将任务分解为两步：第一步，检索：在知识库中找到与问题相关性最强的片段。第二步，生成：基于找到的证据流畅的组织语言。
Traditionelle Modelle haben die Aufgabe, das nächste wahrscheinlichste Wort vorherzusagen. RAG teilt die Aufgabe in zwei Schritte. Schritt eins: Abruf.
